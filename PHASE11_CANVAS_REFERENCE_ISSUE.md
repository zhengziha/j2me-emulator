# Phase 11: Canvas引用问题调查

## 🎯 目标
解决Canvas对象引用为假引用（0x87654321）的问题

## 🔍 问题分析

### 观察到的现象
1. **对象创建成功**: 类`j`的对象被成功创建，引用为0x2
2. **Canvas引用错误**: Display.setCurrent()接收到的Canvas参数是0x87654321（假引用）
3. **引用丢失**: 真实的对象引用0x2没有被传递到Display.setCurrent()

### 执行流程
```
1. NEW j -> 创建对象 0x2
2. invokespecial j.<init> -> 初始化对象
3. invokestatic XMIDlet.a() -> 静态方法调用
4. invokestatic y.a() -> 静态方法调用
5. Display.setCurrent(0x87654321) -> Canvas引用错误！
```

### 假引用来源
0x87654321这个值可能来自：
1. **getstatic指令**: 当静态字段访问失败时返回的默认值
2. **getfield指令**: 当实例字段访问失败时返回的默认值
3. **字段访问系统**: `j2me_field_access.c`中的默认值

## ✅ 已完成的修复

### 1. 移除假引用返回
**修改文件**: 
- `src/core/j2me_field_access.c`
- `src/interpreter/j2me_interpreter.c`

**修改内容**:
- 将所有假引用（0x87654321, 0x11223344, 0x12345678）改为返回0（null）
- 简化字段访问逻辑，不再使用"智能默认值"

### 2. NEW指令完善
**修改文件**: `src/interpreter/j2me_interpreter.c`

**修改内容**:
- NEW指令现在在堆上创建真实对象
- 对象引用从1开始递增（0x1, 0x2, 0x3...）
- 支持未找到的系统类创建通用对象

## ⚠️ 当前问题

### Canvas引用仍然是假引用
即使修改了所有返回假引用的地方，Canvas引用仍然是0x87654321

**可能原因**:
1. **putfield/putstatic**: 对象引用被存储到字段时可能被替换
2. **方法调用**: 在方法调用过程中对象引用可能丢失
3. **栈操作**: 对象引用在栈操作中可能被错误处理
4. **常量池**: 可能从常量池中读取了错误的值

## 📝 技术笔记

### 对象引用系统
- **堆引用**: 从1开始递增（0x1, 0x2, 0x3...）
- **null引用**: 0
- **假引用**: 0x12345678, 0x87654321, 0x11223344（应该被消除）

### 字段访问问题
当前字段访问是简化实现：
- 不存储实际的字段值
- 总是返回默认值或0
- 这导致对象引用无法通过字段传递

### 可能的根本原因
游戏代码可能是这样的：
```java
Canvas canvas = new j();  // 创建Canvas对象
this.canvas = canvas;     // 存储到字段（putfield）
Display.setCurrent(this.canvas);  // 从字段读取（getfield）
```

如果putfield不存储值，getfield就会返回0或假引用！
