# 交互式渲染测试说明

## 测试目的

通过你的按键反馈，诊断SDL渲染是否正常工作，以及问题出在哪里。

## 运行测试

```bash
./build/interactive_render_test
```

## 测试流程

### 1. 启动程序

程序启动后会显示说明，按 **SPACE** 开始测试。

### 2. 测试项目

程序会依次显示5个测试画面，每个测试后请根据你看到的内容按键：

#### 测试1: 全屏红色
- **预期**: 窗口显示纯红色背景
- **按键**:
  - `1` = 看到红色 ✓
  - `0` = 看到其他颜色或黑色 ✗
  - `n` = 完全看不到窗口 ✗

#### 测试2: 全屏绿色
- **预期**: 窗口显示纯绿色背景
- **按键**:
  - `1` = 看到绿色 ✓
  - `0` = 看到其他颜色或黑色 ✗
  - `n` = 完全看不到窗口 ✗

#### 测试3: 全屏蓝色
- **预期**: 窗口显示纯蓝色背景
- **按键**:
  - `1` = 看到蓝色 ✓
  - `0` = 看到其他颜色或黑色 ✗
  - `n` = 完全看不到窗口 ✗

#### 测试4: 黑色背景 + 白色矩形
- **预期**: 窗口显示黑色背景，中间有一个白色矩形
- **按键**:
  - `1` = 看到白色矩形 ✓
  - `0` = 只看到黑色或其他 ✗
  - `n` = 完全看不到窗口 ✗

#### 测试5: 多个彩色矩形
- **预期**: 窗口显示白色背景，上面有多个不同颜色的矩形（红、绿、蓝、黄、青、品红）
- **按键**:
  - `1` = 看到多个彩色矩形 ✓
  - `0` = 只看到单色或其他 ✗
  - `n` = 完全看不到窗口 ✗

### 3. 查看结果

所有测试完成后，程序会显示：
- 测试结果总结
- 成功/失败统计
- 诊断建议

### 4. 退出

按 **ESC** 随时退出测试。

## 结果解读

### 情况1: 所有测试通过（5个成功）
```
✓ SDL渲染完全正常！
```
**结论**: SDL本身工作正常，问题在于J2ME的canvas texture渲染。

**下一步**:
- 检查canvas texture的创建参数
- 验证渲染目标设置
- 检查display_refresh的实现

### 情况2: 看不到窗口（多个'n'）
```
✗ SDL窗口不可见！
```
**可能原因**:
1. 窗口在其他桌面/工作区（macOS的Mission Control）
2. 窗口被其他窗口完全遮挡
3. 窗口创建在屏幕外
4. SDL配置问题

**下一步**:
- 检查是否有多个桌面
- 尝试最小化其他窗口
- 修改窗口创建位置
- 添加窗口提升代码

### 情况3: 看到窗口但内容错误（多个'0'）
```
✗ SDL渲染有问题！
```
**可能原因**:
1. 颜色格式问题
2. 渲染器配置问题
3. 显卡驱动问题
4. SDL版本不兼容

**下一步**:
- 检查SDL版本
- 尝试不同的渲染器标志
- 更新显卡驱动
- 尝试软件渲染器

### 情况4: 部分测试通过
```
部分测试通过
```
**分析**:
- 如果简单测试（1-3）通过，复杂测试（4-5）失败 → 可能是图形绘制API问题
- 如果随机失败 → 可能是时序或刷新问题

## 注意事项

1. **窗口位置**: 窗口会在屏幕中央创建，大小为240x320像素
2. **窗口标题**: "SDL渲染测试 - 请观察窗口内容"
3. **测试时间**: 每个测试没有时间限制，按键后才会进入下一个测试
4. **重复测试**: 如需重复测试，重新运行程序即可

## 故障排除

### 问题: 程序启动后立即退出
**解决**: 检查SDL是否正确安装
```bash
pkg-config --modversion sdl2
```

### 问题: 找不到窗口
**解决**: 
1. 检查所有桌面/工作区（macOS: F3或三指上滑）
2. 尝试Command+Tab切换窗口
3. 检查Dock中是否有程序图标

### 问题: 窗口显示但是黑色
**解决**: 这是重要信息！请在测试中按'0'记录

## 下一步

根据测试结果，我们将：
1. 如果SDL正常 → 修复J2ME canvas texture渲染
2. 如果SDL有问题 → 调整SDL配置或寻找替代方案
3. 如果看不到窗口 → 修改窗口创建参数
